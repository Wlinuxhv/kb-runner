# 执行 ID 格式说明

## 格式定义

```
exec-YYYYMMDD-HHMMSS-ffffff
```

## 示例

```
exec-20260327-094053-900583
```

## 字段说明

| 字段 | 含义 | 示例 | 说明 |
|------|------|------|------|
| `exec` | 前缀 | exec | 表示执行 ID |
| `YYYYMMDD` | 日期 | 20260327 | 2026 年 03 月 27 日 |
| `HHMMSS` | 时间 | 094053 | 09:40:53 |
| `ffffff` | 微秒 | 900583 | 微秒级别（000000-999999） |

## 时间解析

**执行 ID**: `exec-20260327-094053-900583`

**对应时间**:
- 日期：2026 年 03 月 27 日
- 时间：09 点 40 分 53 秒
- 微秒：900583 微秒

**Unix 时间戳**: 1774575653

## 为什么使用这种格式？

### 优点

1. **可读性强** - 一眼就能看出执行时间
2. **排序方便** - 按文件名排序即按时间排序
3. **唯一性保证** - 微秒部分保证同一秒内的唯一性
4. **调试友好** - 日志分析时容易关联

### 对比旧格式

**旧格式**: `exec-1774575653-900583`
- 优点：简短
- 缺点：时间不直观，需要转换

**新格式**: `exec-20260327-094053-900583`
- 优点：时间直观，易于阅读和调试
- 缺点：稍长（增加约 10 个字符）
