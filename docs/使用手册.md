## 简介

KB Runner 是一个 KB 脚本执行框架，用于执行 KB 检查脚本并生成结果矩阵。

### 主要特性

- ✅ 纯 Bash 实现，无 Python 依赖
- ✅ 支持配置文件定义步骤权重
- ✅ 自动得分计算（满分 100 分）
- ✅ 支持批量执行和排名
- ✅ Offline 模式分析日志包
- ✅ 结果文件 JSON 格式

## 快速开始

### 1. 执行单个 KB

```bash
# 通过统一入口执行
./kbscript/run.sh <KB 名称>

# 示例
./kbscript/run.sh 示例 KB-00001
```

### 2. 批量执行所有 KB

```bash
# 执行所有 KB
./kbscript/run_all.sh

# 执行指定的 KB
./kbscript/run_all.sh 示例 KB-00001 A100 显卡问题 -32920
```

### 3. 使用 CLI 工具

```bash
# 执行所有 KB
./bin/kb-runner run --all

# 按分类执行
./bin/kb-runner run --category hardware

# 按标签执行
./bin/kb-runner run --tags gpu,nvidia

# 执行指定 KB 列表
./bin/kb-runner run --kb 00001,32920,35871
```

## CLI 命令

### kb-runner run

执行 KB 脚本。

```bash
kb-runner run [选项]
```

#### 选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--Q <Q 单号>` | Q 单号（offline 模式必需） | - |
| `--all` | 执行所有 KB 脚本 | false |
| `--kb <列表>` | 执行指定的 KB 列表（逗号分隔） | - |
| `--category <类别>` | 执行指定分类的所有 KB | - |
| `--tags <标签>` | 执行指定标签的所有 KB | - |
| `--preprocess` | 启用日志预处理（默认启用） | true |
| `--unpreprocesslog` | 禁用日志预处理 | false |
| `--processed-path <路径>` | 已处理的日志路径 | - |
| `--result-dir <目录>` | 结果输出目录（覆盖配置） | - |
| `--log-password <密码>` | 日志包密码（覆盖默认） | - |
| `--case <名称>` | 按 CASE 名称执行 | - |
| `--scenario <名称>` | 按场景执行 | - |
| `--interactive` | 交互式选择执行 | false |
| `--mode <模式>` | 运行模式 (online/offline) | online |
| `--qno <Q 单号>` | 离线日志包 Q 单号 | - |
| `--log-root <路径>` | 离线日志根目录 | ./workspace/icare_log/logall |

#### 示例

```bash
# Online 模式执行单个脚本
kb-runner run -s ./scripts/check.sh -l bash

# Offline 模式执行所有 KB
kb-runner run --Q Q2026031700281 --all

# 执行指定分类
kb-runner run --category hardware --all

# 使用已解压的日志
kb-runner run --Q Q2026031700281 --unpreprocesslog --processed-path /data/logs/Q2026031700281 --all
```

## 编写 KB 脚本

### 1. 目录结构

```
kbscript/
└── KB 名称-ID/
    ├── case.yaml          # 配置文件
    └── run.sh             # 执行脚本
```

### 2. 配置文件 (case.yaml)

```yaml
name: "KB 名称"
language: bash
category: hardware
tags:
  - gpu
  - nvidia
description: "KB 描述"
timeout: 300s

# 得分配置
scoring:
  max_score: 100.0
  steps:
    - name: "步骤 1 名称"
      weight: 0.3          # 权重（占总分的 30%）
      expected_status: "success"  # 期望状态
      description: "步骤描述"
    
    - name: "步骤 2 名称"
      weight: 0.4
      expected_status: "success"
    
    - name: "步骤 3 名称"
      weight: 0.3
      expected_status: "success"

params:
  kb_id: "00001"

extensions:
  author: "team-name"
  version: "1.0.0"
```

### 3. 执行脚本 (run.sh)

```bash
#!/bin/bash
# CASE: KB 名称

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR" && cd ../.. && pwd)"

source "$PROJECT_ROOT/scripts/bash/api.sh"

kb_init

# 步骤 1
step_start "步骤 1 名称"
# 检查逻辑...
if [ 检查通过 ]; then
    result "key" "value"
    step_success "步骤成功消息"
else
    step_failure "步骤失败消息"
fi

# 步骤 2
step_start "步骤 2 名称"
# ...

kb_save
```

### 4. API 函数

#### 初始化
- `kb_init()` - 初始化结果文件
- `kb_save()` - 保存最终结果

#### 日志
- `log_debug "消息"`
- `log_info "消息"`
- `log_warn "消息"`
- `log_error "消息"`

#### 步骤
- `step_start "名称"` - 开始步骤
- `step_success "消息"` - 步骤成功
- `step_failure "消息"` - 步骤失败
- `step_warning "消息"` - 步骤警告
- `step_skip "消息"` - 跳过步骤
- `step_output "内容"` - 设置步骤输出

#### 结果
- `result "key" "value"` - 记录键值对结果

#### Offline 模式
- `kb_offline_log_dir()` - 获取日志目录
- `kb_offline_host()` - 获取当前主机
- `kb_exec "命令"` - 受控执行命令（offline 模式禁止）

## Offline 模式

### 1. 自动预处理日志包

```bash
# 自动解压日志包并执行 KB
kb-runner run --Q Q2026031700281 --all
```

### 2. 使用已解压的日志

```bash
# 禁用预处理，使用已解压的目录
kb-runner run --Q Q2026031700281 --unpreprocesslog \
  --processed-path /data/kb-runner/workspace/icare_log/logall/2603/Q2026031700281 \
  --all
```

### 3. 使用统一入口脚本

```bash
# Offline 模式执行单个 KB
./kbscript/run.sh 示例 KB-00001 --offline --qno Q2026031700281

# 指定日志根目录和主机
./kbscript/run.sh 示例 KB-00001 --offline \
  --qno Q2026031700281 \
  --log-root /data/logs \
  --host sp_10.56.194.139
```

## 常见问题

### 1. 步骤没有保存

确保：
- 正确 source api.sh
- 使用 `step_start` 开始步骤
- 使用 `step_success/failure` 结束步骤
- 最后调用 `kb_save`

### 2. 得分计算不正确

检查：
- case.yaml 中步骤名称是否与脚本中一致
- 权重总和是否为 1.0
- 期望状态是否正确配置

### 3. Offline 模式无法解压日志

确保：
- 日志包路径正确
- 系统安装了 unzip 命令
- 日志包密码正确
