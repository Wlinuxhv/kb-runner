# KB Runner 重构说明文档

## 概述

本次重构实现了基于配置文件的 KB 脚本得分计算系统，支持每个 KB 独立配置步骤权重和期望状态，最终生成带排名信息的 JSON 结果文件。

## 主要变更

### 1. 配置文件结构 (case.yaml)

每个 KB 脚本对应一个 `case.yaml` 配置文件，包含：

```yaml
name: "KB 名称"
language: bash
category: 类别
tags:
  - 标签 1
  - 标签 2
description: "描述"
timeout: 300s

# 得分配置 - 满分 100 分
scoring:
  max_score: 100.0
  
  # 步骤配置：每个步骤的权重和期望状态
  steps:
    - name: "步骤 1 名称"
      weight: 0.3         # 权重 (占总分的 30%)
      expected_status: "success"  # 期望状态：success 或 failure
      description: "步骤描述"
      
    - name: "步骤 2 名称"
      weight: 0.4         # 权重 (占总分的 40%)
      expected_status: "success"
      description: "步骤描述"
      
    - name: "步骤 3 名称"
      weight: 0.3         # 权重 (占总分的 30%)
      expected_status: "success"
      description: "步骤描述"

params:
  kb_id: "00001"

extensions:
  author: "team-name"
  version: "1.0.0"
  custom_fields:
    priority: "high"
```

### 2. Bash API 变更 (`scripts/bash/api.sh`)

#### 新增环境变量
- `KB_CONFIG_FILE`: 配置文件路径
- `KB_MAX_SCORE`: 满分分数 (默认 100.0)
- `KB_SCRIPT_NAME`: 脚本名称

#### 新增功能
- `_kb_load_step_config()`: 从配置文件加载步骤权重和期望状态
- `_kb_calculate_score()`: 根据配置计算得分
- 步骤结果包含：weight, expected_status, score, max_score

#### 得分计算规则
- 步骤状态与期望状态匹配：获得该步骤的权重分
- 步骤状态与期望状态不匹配：得 0 分
- 最终得分 = (总得分 / 总权重) × max_score

### 3. 统一入口脚本 (`kbscript/run.sh`)

所有 KB 脚本通过统一入口执行：

```bash
# 执行单个 KB
./kbscript/run.sh <kb_name>

# 示例
./kbscript/run.sh test_case
./kbscript/run.sh "A100 显卡问题 -32920"
```

### 4. 批量执行脚本 (`kbscript/run_all.sh`)

批量执行多个 KB 并生成带排名的结果：

```bash
# 执行所有 KB
./kbscript/run_all.sh

# 执行指定的 KB
./kbscript/run_all.sh test_case "A100 显卡问题 -32920"
```

## 使用示例

### 创建新的 KB 脚本

1. 创建目录：
```bash
mkdir kbscript/my_kb-00001
```

2. 创建配置文件 `case.yaml`：
```yaml
name: "我的 KB"
language: bash
category: custom
description: "我的自定义 KB 脚本"
timeout: 300s

scoring:
  max_score: 100.0
  steps:
    - name: "检查环境"
      weight: 0.4
      expected_status: "success"
    - name: "执行测试"
      weight: 0.6
      expected_status: "success"

params:
  kb_id: "00001"
```

3. 创建脚本 `run.sh`：
```bash
#!/bin/bash
source "$PROJECT_ROOT/scripts/bash/api.sh"

kb_init

step_start "检查环境"
# 检查逻辑...
step_success "环境检查通过"

step_start "执行测试"
# 测试逻辑...
step_success "测试通过"

kb_save
```

4. 执行：
```bash
./kbscript/run.sh my_kb-00001
```

### 批量执行并查看排名

```bash
./kbscript/run_all.sh
```

输出示例：
```

排名  KB 名称                                    得分        状态
```

## 配置说明

### 权重配置

- 权重总和不需要严格为 1.0，系统会自动归一化
- 建议权重总和为 1.0，便于理解每个步骤的占比

### 期望状态

- `success`: 期望步骤成功（默认）
- `failure`: 期望步骤失败（用于测试异常场景）

### 扩展字段

`extensions` 字段可以包含任何自定义信息：
- author: 作者
- version: 版本号
- created_at: 创建时间
- related_kbs: 相关 KB 列表
- 任何自定义字段

## 迁移指南

### 从旧版本迁移

1. 为现有 KB 脚本创建 `case.yaml` 配置文件
2. 在脚本开头添加环境变量导出
3. 使用统一入口脚本执行

### 兼容性

- 旧的脚本仍然可以运行，但不会使用新的得分计算
- 建议逐步迁移到新的配置驱动模式

```
kb-runner/
├── scripts/
│   └── bash/

├── kbscript/

│   ├── 示例 KB-00001/          # KB 目录

│       ├── case.yaml
│       └── run.sh

├── temp/                       # 结果文件目录
└── logs/                       # 日志文件目录
```

## 注意事项

1. **Python3 依赖**: 新的得分计算需要 Python3 和 PyYAML
2. **配置文件编码**: 使用 UTF-8 编码
3. **步骤名称匹配**: 脚本中的 step_start 名称必须与配置文件中的 name 完全匹配
4. **权限**: 确保 run.sh 有执行权限 (chmod +x)
